# Changelog Policy

## Scope
This policy defines how changes are recorded for releases.

## Format

- Use docs/CHANGELOG.md.
- Group changes by release version.
- Use the following categories:
  - Added
  - Changed
  - Fixed
  - Deprecated
  - Removed
  - Security

## Versioning

Versions follow semantic versioning.
Breaking changes require a major version increment.

## Required entries

- Each release must include a summary of changes.
- Security relevant changes must be listed under Security.
